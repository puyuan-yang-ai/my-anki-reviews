以下是前後端分開啟動的梳理（按順序操作即可）：

後端（FastAPI，預設埠 9006）
- 環境：建議 Python 3.12（Dockerfile 也是 3.12 基底），先在 `project_code/career` 下建立虛擬環境並安裝依賴 `pip install -r requirements.txt`。
- 配置：`code/config/config.yaml` 內含 DB / Redis / AI Key 等遠端地址與憑證，若本機不可訪問或不想用線上資源，請改成本地/測試配置（也避免把敏感值提交到版本庫）。
- 啟動：
  ```bash
  cd project_code/career/code
  # 方式1：直接跑腳本
  python app.py
  # 方式2：開發熱重載（需 uvicorn 已安裝）
  uvicorn app:app --host 0.0.0.0 --port 9006 --reload
  ```
- 成功後檢查健康：`http://localhost:9006/resume/deep-optimization/converse`（POST，供前端串流對話）與 `.../generate`（POST，生成簡歷）。

前端（Gradio，預設埠 7860）
- 環境：在 `project_code/career/frontend` 執行 `pip install -r requirements.txt`。
- 依賴後端：預設指向 `http://localhost:9006`（見 `frontend/config.py` 及 `frontend/app.py` 內的 `API_BASE_URL`）。若後端不在本機或端口不同，請改這個值。
- 啟動：
  ```bash
  cd project_code/career/frontend
  python app.py
  ```
- 存取：瀏覽器開 `http://localhost:7860`。開始對話前請先確保後端已起且能訪問 9006 埠。

最小驗證流程
- 先啟後端（確保 9006 通），再啟前端（7860）。
- 在前端點「開始對話」測試 `/resume/deep-optimization/converse`；對話結束後點「生成簡歷」測試 `/resume/deep-optimization/generate`。

附加提醒
- `config.yaml` 中的外部憑證與地址務必妥善保護，建議改為環境變數或本地測試值。
